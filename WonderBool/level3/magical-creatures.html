Enchanted Classification Tree (BST)

Binary Search Tree (Arbor Cognitus)

A Binary Search Tree (BST) is a magical data structure where each creature node has at most two children (left and right). Creatures are organized based on their identifier:

  • Identifiers less than the parent node go to the left child.
  • Identifiers greater than the parent node go to the right child.

This structure allows for efficient searching and insertion. However, if creatures are added in a specific order (e.g., sorted), the tree can become unbalanced, degrading performance.

Search/Insert Complexity: Average O(log n), Worst O(n)

An AVL Tree is a self-balancing Binary Search Tree. It maintains the BST property but adds a crucial constraint: the heights of the two child subtrees of any node must differ by at most one. If adding or removing a creature violates this 'balance factor', the tree performs magical rotations (LL, RR, LR, RL) to restore balance.

This balancing act ensures that the tree remains relatively balanced, guaranteeing efficient operations even in the worst case.

Search/Insert Complexity: O(log n) (Worst-case guaranteed)
